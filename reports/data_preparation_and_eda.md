# Подготовка данных и EDA. Подготовка данных и EDA

Этот файл описывает, что именно делается на втором шаге проекта. Конкретные численные результаты, таблицы и графики будут созданы после локального запуска ноутбука или скрипта, потому что исходный файл `household_power_consumption.txt` не хранится в архиве проекта из-за большого размера.

## Цель шага

Цель шага — превратить исходный минутный датасет в подготовленный часовой временной ряд, пригодный для дальнейшего анализа, поиска аномалий и обучения моделей прогнозирования.

## Входные данные

Исходный файл:

```text
data/raw/household_power_consumption.txt
```

Особенности исходного файла:

- разделитель колонок — `;`;
- дата и время находятся в разных колонках: `Date` и `Time`;
- формат даты — `dd/mm/yyyy`;
- пропуски обозначены символом `?`;
- целевая переменная — `Global_active_power`.

## Выходные данные

После выполнения шага создается подготовленный файл:

```text
data/processed/prepared_time_series.csv
```

Формат подготовленного файла:

| Колонка | Описание |
|---|---|
| `unique_id` | идентификатор временного ряда, для основной задачи `household_1` |
| `ds` | временная метка с шагом 1 час |
| `y` | среднее часовое значение `Global_active_power` |

## Логика предобработки

1. Загрузить исходный файл `household_power_consumption.txt`.
2. Объединить колонки `Date` и `Time` в одну колонку `ds`.
3. Преобразовать `ds` к типу `datetime`.
4. Преобразовать числовые признаки к числовому типу.
5. Символы `?` преобразовать в `NaN`.
6. Отсортировать данные по времени.
7. Удалить дубликаты временных меток, если они есть.
8. Агрегировать минутные наблюдения до часовых.
9. Для целевой переменной использовать среднее значение за час.
10. Привести ряд к регулярной часовой сетке.
11. Заполнить пропуски после агрегации интерполяцией по времени.
12. Сохранить подготовленный ряд в `data/processed/prepared_time_series.csv`.

## Почему используется среднее за час

Целевая переменная `Global_active_power` измеряется в киловаттах. Поэтому при переходе от минутной частоты к часовой используется среднее значение за час, а не сумма. Такой подход сохраняет исходный смысл переменной: средний уровень активной мощности за период.

## Что входит в EDA

В EDA включены следующие проверки и графики:

1. Проверка количества строк, дат, пропусков и дубликатов.
2. Проверка типов данных.
3. Проверка равномерности временного шага.
4. Описательная статистика целевой переменной.
5. Общий график подготовленного ряда.
6. График последних 60 дней.
7. Скользящие средние с окнами 24 часа и 168 часов.
8. Средний профиль по часу суток.
9. Средний профиль по дню недели.
10. Средний профиль по месяцу.
11. Декомпозиция последних 90 дней на тренд, сезонность и остатки.
12. ADF и KPSS тесты на стационарность.
13. ACF/PACF для анализа автокорреляции.

## Ожидаемый вывод для отчета

После подготовки данных ряд должен быть регулярным часовым временным рядом без пропусков в целевой переменной. Ожидается наличие суточной сезонности, поэтому в дальнейших моделях базовый сезонный период задается как `24`. Для недельных эффектов используется дополнительный период `168`.
