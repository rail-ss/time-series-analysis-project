# Выбор датасета и постановка задачи. Выбор временного ряда и постановка задачи

## 1. Выбранный датасет

Для итогового задания выбран датасет **Individual Household Electric Power Consumption** из UCI Machine Learning Repository.

**Почему это хороший выбор для работы:**

- это реальный временной ряд, а не искусственный пример;
- данные достаточно длинные для обучения и тестирования разных моделей;
- есть пропуски, аномалии и шум, поэтому можно показать полноценную предобработку;
- есть ожидаемая суточная и недельная сезонность;
- датасет подходит и для статистических моделей, и для ML/DL-подходов.

## 2. Что прогнозируем

Прогнозируем среднее часовое значение активной мощности домохозяйства:

```text
y = среднее значение Global_active_power за час
```

Целевая переменная в исходных данных — `Global_active_power`, то есть глобальная активная мощность домохозяйства в киловаттах.

## 3. Горизонт прогнозирования

**Горизонт прогноза:** 24 часа вперед.

Обоснование: это краткосрочный прогноз на сутки вперед. Такой горизонт хорошо подходит для анализа электропотребления, потому что внутри суток обычно проявляется важная сезонная структура: утренние, дневные и вечерние пики.

## 4. Частота данных

**Исходная частота:** 1 минута.  
**Рабочая частота:** 1 час.

Исходные минутные данные агрегируются до часовых значений по среднему. Это уменьшает шум, ускоряет обучение моделей и сохраняет суточную сезонность.

## 5. Одна серия или много

В основной постановке используется одна временная серия:

```text
unique_id = household_1
```

Это ряд электропотребления одного домохозяйства. Глобальные модели не обязательны, потому что нет набора из многих независимых рядов. Однако формат `unique_id`, `ds`, `y` сохраняется, чтобы проект был совместим с `statsforecast`, `mlforecast` и `neuralforecast`.

## 6. Offline или online

Задача решается как **offline-прогнозирование**.

Это значит, что модель обучается на исторических данных, затем проверяется на отложенном временном интервале. Online-переобучение после каждого нового наблюдения в работе не требуется.

## 7. Метрики качества

В работе будут использоваться следующие метрики:

| Метрика | Что показывает | Почему нужна |
|---|---|---|
| MAE | Среднюю абсолютную ошибку в kW | Простая интерпретация ошибки |
| RMSE | Ошибку с усиленным штрафом за крупные промахи | Важна для пиков потребления |
| MAPE | Среднюю процентную ошибку | Дополнительная интерпретация в процентах |
| sMAPE | Симметричную процентную ошибку | Более устойчива, чем MAPE |
| MASE | Ошибку относительно naive baseline | Позволяет понять, лучше ли модель простого прогноза |

Основными метриками для выбора модели будут **MAE, RMSE, sMAPE и MASE**. MAPE будет использоваться осторожно, потому что при очень малых значениях потребления она может давать искаженные результаты.

## 8. Финальная формулировка задачи

В рамках итоговой работы решается задача краткосрочного прогнозирования временного ряда электропотребления одного домохозяйства. На основе исторических часовых значений `Global_active_power` необходимо построить прогноз на следующие 24 часа, сравнить baseline, статистические модели, ML-модели и DL-модели, а затем выбрать наилучшую модель по набору метрик MAE, RMSE, sMAPE и MASE.

## 9. Что будет сделано на следующем шаге

Следующий шаг — подготовка данных и EDA:

1. загрузить датасет;
2. преобразовать дату и время;
3. проверить пропуски и дубликаты;
4. агрегировать данные до часовой частоты;
5. построить первые графики;
6. сохранить подготовленный ряд.
